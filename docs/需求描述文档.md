# Nova-X 全球智能能源基础设施平台 —— 需求描述文档

## 文档信息
- **项目名称：** Nova-X (星云) 全球智能能源基础设施与微电网调度平台
- **文档版本：** v1.0
- **创建日期：** 2026-01-20
- **适用终端：** Web端、小程序端、APP端

---

## 1. 项目概述

### 1.1 产品定位
Nova-X 是一个全球领先的能源互联网中枢系统，通过数字化手段连接碎片化的新能源资产，实现能源在生产、存储与消耗环节的高效调度。平台需支持多终端接入，为不同角色提供差异化的服务体验。

### 1.2 核心价值
- 打破能源孤岛，实现电网负荷平衡
- 提升绿色能源利用率
- 构建可自我修复、智能预测的全球补能网络
- 为用户提供跨平台、一致性的使用体验

## 4. 非功能性需求

### 4.1 性能要求

#### 4.1.1 并发处理能力
- **核心链路 TPS：** ≥ 50,000
- **扫码启动响应时间：** ≤ 500ms
- **地图查询响应时间：** ≤ 300ms
- **支付确认时间：** ≤ 2s
- **实时数据延迟：** ≤ 1s

#### 4.1.2 数据处理能力
- **设备上报频率：** 每 5 秒/次
- **日数据处理量：** ≥ 10 亿条
- **数据压缩率：** ≥ 70%
- **历史数据查询：** ≤ 3s（近 1 年数据）

### 4.2 可用性要求
- **系统 SLA：** 99.99%
- **故障恢复时间 (RTO)：** ≤ 5 分钟
- **数据丢失容忍度 (RPO)：** ≤ 1 分钟
- **异地多活：** 至少 3 个数据中心
- **自动故障切换：** 业务无感知

### 4.3 安全性要求

#### 4.3.1 数据安全
- **数据传输加密：** TLS 1.3
- **敏感数据加密：** AES-256
- **数据脱敏：** 个人信息、支付信息

- **防篡改：** 关键业务数据区块链存证

#### 4.3.2 身份认证
- **多因素认证 (MFA)：** 管理端强制启用
- **OAuth 2.0 / OpenID Connect**
- **Token 刷新机制：** 短期有效 + 自动刷新
- **设备指纹识别**
- **风险行为检测**

#### 4.3.3 支付安全
- **PCI DSS 合规**
- **实名认证强制绑定**
- **大额交易二次确认**
- **资金流水防篡改**

## 6. 核心业务流程

### 6.1 用户充电流程

```
1. 用户打开 APP/小程序
2. 定位并查找附近充电站
3. 选择目标站点，查看桩位状态
4. 导航至站点
5. 扫描充电桩二维码 / NFC 识别
6. 系统鉴权，预扣费
7. 下发启动指令至充电桩
8. 充电桩确认启动，开始充电
9. 实时展示充电进度
10. 充电完成或用户主动停止
11. 下发停止指令
12. 系统结算费用
13. 推送订单详情
14. 用户评价与反馈
```

### 6.2 运营商设备管理流程

```
1. 添加充电站信息
2. 绑定充电设备
3. 配置设备参数（功率、价格、协议）
4. 设备上线，开始接受订单
5. 实时监控设备状态
6. 接收异常告警
7. 派发维修工单
8. 设备维护完成，恢复服务
9. 查看营收报表
10. 调整定价策略
```

### 6.3 需求响应流程

```
1. 电力供应商监测电网负荷
2. 预测负荷超限风险
3. 创建 DR 指令
4. 选择响应区域与设备
5. 下发功率限制指令
6. 平台接收指令，执行调度策略
7. 动态调整各充电桩输出功率
8. 实时监控执行效果
9. 负荷恢复正常
10. 计算补偿收益
11. 结算与分账
```

## 7. 数据字典（核心实体）

### 7.1 用户 (User)
- user_id: 用户唯一标识
- phone: 手机号
- email: 邮箱
- nickname: 昵称
- avatar: 头像
- real_name: 实名
- id_card: 身份证号（加密）
- registration_time: 注册时间
- last_login_time: 最后登录时间

### 7.2 车辆 (Vehicle)
- vehicle_id: 车辆标识

- model: 车型
- battery_capacity: 电池容量 (kWh)

### 7.3 充电站 (Station)
- station_id: 站点标识
- station_name: 站点名称
- operator_id: 运营商 ID
- longitude: 经度
- latitude: 纬度
- address: 详细地址
- total_connectors: 总枪口数
- available_connectors: 可用枪口数
- facilities: 设施信息（JSON）
- business_hours: 营业时间
- status: 站点状态（营业中、维护中、停用）

### 7.4 充电设备 (Charger)
- charger_id: 设备标识
- station_id: 所属站点
- manufacturer: 制造商
- model: 设备型号
- serial_number: 序列号
- firmware_version: 固件版本
- protocol: 通信协议
- max_power: 最大功率 (kW)
- status: 设备状态（在线、离线、故障）
- last_heartbeat: 最后心跳时间

### 7.5 充电枪口 (Connector)
- connector_id: 枪口标识
- charger_id: 所属设备
- connector_type: 接口类型
- max_current: 最大电流 (A)
- max_voltage: 最大电压 (V)
- status: 枪口状态（空闲、充电中、故障、预约中）

### 7.6 充电订单 (ChargingSession)
- session_id: 订单标识
- user_id: 用户 ID
- connector_id: 枪口 ID
- start_time: 开始时间
- end_time: 结束时间
- energy: 充电电量 (kWh)
- duration: 充电时长 (min)
- amount: 订单金额
- payment_method: 支付方式
- status: 订单状态（进行中、已完成、已取消、异常）

### 7.7 价格策略 (PricingRule)
- rule_id: 规则标识
- station_id: 适用站点

- end_time: 生效时间段结束
- electricity_price: 电费单价 (元/kWh)
- service_fee: 服务费 (元/kWh)
- is_dynamic: 是否动态定价

### 7.8 需求响应指令 (DemandResponse)
- dr_id: 指令标识
- issuer_id: 发布方
- target_region: 目标区域
- power_limit: 功率限制 (kW)
- start_time: 开始时间
- end_time: 结束时间
- compensation_rate: 补偿费率
- status: 执行状态

## 8. 接口设计规范

### 8.1 RESTful API 规范
- **统一前缀：** /api/v1
- **命名规范：** 小写字母 + 下划线
- **HTTP 方法：** GET (查询), POST (创建), PUT (更新), DELETE (删除)
- **状态码：** 200 (成功), 400 (参数错误), 401 (未授权), 403 (禁止访问), 404 (不存在), 500 (服务器错误)

### 8.2 响应格式
```json
{
  "code": 200,
  "message": "success",
  "data": {},
  "timestamp": 1642665600000,
  "trace_id": "abc123def456"
}
```

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "list": [],
    "total": 100,
    "page": 1,
    "page_size": 20
  }
}
```
